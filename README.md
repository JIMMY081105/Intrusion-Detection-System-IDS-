# Metaheuristic Feature Selection for Network Intrusion Detection

## Description

This project implements and compares three metaheuristic feature selection algorithms — Genetic Algorithm (GA), Particle Swarm Optimization (PSO), and Simulated Annealing (SA) — against a Random Forest baseline for multi-class network intrusion detection on the **CICIDS2017** dataset. Both raw features and PCA-reduced features are evaluated to measure the effect of dimensionality reduction on each method.

---

## Objectives

1. Preprocess the CICIDS2017 dataset: merge, clean, deduplicate, scale, and apply PCA.
2. Establish a Random Forest baseline on raw features and PCA-reduced features.
3. Apply GA, PSO, and SA to identify optimal feature subsets via a validation-based search.
4. Compare methods using accuracy, macro-precision, macro-recall, macro-F1, balanced accuracy, and runtime.
5. Analyse train-test generalisation gaps and per-class confusion matrix statistics.

## Dataset Setup

This project uses the **CICIDS2017** (Canadian Institute for Cybersecurity Intrusion Detection System 2017) dataset.

### Steps

1. Download the dataset from the official source:
   [https://www.unb.ca/cic/datasets/ids-2017.html](https://www.unb.ca/cic/datasets/ids-2017.html)

2. Place **all raw CSV files** into the `data/raw/` directory:

```
data/

    ├── Tuesday-WorkingHours.pcap_ISCX.csv
    ├── Wednesday-workingHours.pcap_ISCX.csv
    ├── Thursday-WorkingHours-Morning-WebAttacks.pcap_ISCX.csv
    ├── Thursday-WorkingHours-Afternoon-Infilteration.pcap_ISCX.csv
    ├── Friday-WorkingHours-Morning.pcap_ISCX.csv
    ├── Friday-WorkingHours-Afternoon-PortScan.pcap_ISCX.csv
    └── Friday-WorkingHours-Afternoon-DDos.pcap_ISCX.csv
```

3. The preprocessing script will merge these files and save all outputs to `data/processed/`.

## Execution Order

All scripts must be run from the **`src/` directory**. Run them in this order:

### Step 1 — Prepare the Dataset

```bash
cd src
python dataset_prepare.py
```

Reads raw CSVs from `data/raw/`, performs:
- Merging all CSV files into one dataset
- Label encoding and non-feature column removal
- Deduplication before train/test split
- Stratified 80/20 train-test split (seed: 42)
- Median imputation and standard scaling (fit on train only)
- PCA retaining 80% of variance (fit on train only)
- Saves audit logs, class distributions, and overlap reports

Outputs to `data/processed/`:
- `train_multiclass.csv`, `test_multiclass.csv`
- `train_pca_multiclass.csv`, `test_pca_multiclass.csv`
- `label_mapping.csv`, `dataset_audit_summary.json`
- Various audit CSV files

### Step 2 — Run the Baseline (Random Forest)

```bash
python random_forest_raw.py
python random_forest_pca.py
```

Each trains a Random Forest (100 trees, seed 42) and evaluates on the test set.

### Step 3 — Run Metaheuristic Feature Selection

Each script runs 6 hyperparameter configurations, selects the best by validation F1, and reports final test metrics.

```bash
# Genetic Algorithm
python genetic_algorithm_raw.py
python genetic_algorithm_pca.py

# Particle Swarm Optimization
python particle_swarm_optimization_raw.py
python particle_swarm_optimization_pca.py

# Simulated Annealing
python simulated_annealing_raw.py
python simulated_annealing_pca.py
```

> Each metaheuristic script takes significant time (~15 generations/iterations × 6 configs × population/swarm evaluation). Expect several hours per script on CPU.

## Method Overview

| Method | Approach | Key Hyperparameters Evaluated |
|--------|----------|-------------------------------|
| **Random Forest (Raw)** | Baseline on all raw features | 100 trees, fixed |
| **Random Forest (PCA)** | Baseline on PCA components | 100 trees, fixed |
| **Genetic Algorithm** | Binary feature masks evolved via tournament selection, crossover, mutation | Population: 20/50; Mutation rate: 0.05/0.10/0.20 |
| **Particle Swarm Optimization** | Swarm of particles updating positions via velocity equations | Swarm: 20/50; Inertia weight: 0.50/0.70/0.90 |
| **Simulated Annealing** | Single solution with probabilistic acceptance of worse neighbours | Temp: 2.0/5.0; Cooling: 0.90/0.95; Flip bits: 1/2 |

All metaheuristics use a Random Forest (100 trees) as the internal fitness evaluator. Feature selection fitness is measured on a validation split (20% of training data, never the test set).

---

## Evaluation Metrics

- **Accuracy** — proportion of correctly classified samples
- **Precision (macro)** — average precision across all traffic classes
- **Recall (macro)** — average recall across all traffic classes
- **F1-score (macro)** — harmonic mean of macro precision and recall
- **Balanced Accuracy** — accounts for class imbalance
- **Per-class FPR / TPR** — false positive rate and true positive rate derived from the confusion matrix
- **Train-test gap** — difference between training and test metrics (overfitting indicator)
- **Runtime** — train time, inference time, and total search time

---

## Reproducibility

- All random seeds are fixed at **42** (`GLOBAL_SEED = 42`, `random_state=42`).
- The train-test split uses stratification to preserve class proportions.
- The same split is used for both raw and PCA datasets.
- Each metaheuristic configuration uses a deterministic per-config seed (`GLOBAL_SEED + config_index`), ensuring independent and reproducible runs.
- Imputation, scaling, and PCA are all fit exclusively on the training set to prevent data leakage.

## Notes for Markers

- **No GPU required.** If cuML is not installed, all scripts fall back to CPU automatically with no code changes needed.
- **W&B is optional.** All scripts run and print full metrics to the terminal regardless of W&B availability.
- **Scripts can be run independently** after `dataset_prepare.py` has been run once.
- **No absolute paths** are used. All paths are resolved relative to the project root using `os.path`.
- **Data leakage prevention** is explicitly implemented: imputation, scaling, PCA, and metaheuristic validation splits are all computed on training data only, never on the test set.
- **Exact train-test overlap detection** is performed in every model script before evaluation.
